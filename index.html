Scope

What UMI is and is not

Precise scope builds more trust than broad claims.

What UMI is
A 128-bit fixed-width encoding of structural event metadata
A queryable trace stored alongside the raw source record — raw text is always preserved
An open standard — any conformant encoder may produce valid UMI records
A format designed for compliance queryability, not compliance determination
Substrate-agnostic: AI outputs, network logs, software events, transactions
What UMI is not
Not a truth verifier — UMI encodes structure, not factual correctness
Not a replacement for the raw record — it is stored alongside it
Not a complete compliance system — regulatory determination stays with qualified humans
Not dependent on MILANA — any conformant encoder produces valid records
Not a SIEM replacement — a structural signal layer that complements existing systems
The database analogy

A database index does not replace the row — it makes the row findable. UMI works the same way. The raw text is always stored alongside the UMI trace. UMI encodes enough structural metadata to make that text findable in milliseconds without re-reading it.

The problem

Why raw logs are not enough

01
Probabilistic Trap

Text requires post-hoc interpretation

When a DORA auditor asks whether an AI produced a materially incorrect risk assessment, raw logs require re-reading thousands of interactions. That interpretation is itself a source of error and delay.

02
Circular Evaluation

LLM-as-judge adds a second layer of uncertainty

Using an LLM to evaluate another LLM adds a second probabilistic layer to a process that requires determinism. An AI opinion about an AI output is not a deterministic compliance record.

03
The Query Problem

Text cannot be searched deterministically

During a 72-hour NIS2 incident window, a security team cannot locate the exact interaction where a system produced a broken output. The evidence exists. It is not queryable.

Commercial implementations built on UMI

Three proprietary components developed by Aliventi sp. z o.o. Each is optional. The UMI standard functions independently of all three.

MILANA
Reference Encoder

A conformant UMI encoder. Reads raw text and produces structured UMI traces. One implementation among potentially many. Enquiries: project_velana@aliventi.eu

Adds access governance controls to UMI records — who may act on a record, under which jurisdiction, subject to which constraints.

MILANA encoding combined with AiQUS governance in a single artifact. For regulated environments requiring audit trail and access governance simultaneously.
